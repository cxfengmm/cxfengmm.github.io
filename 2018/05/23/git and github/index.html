git和github

git和github的区别

git是版本控制工具

github是网站 ：社交平台、开源项目、远程仓库

svn和git的区别

svn为集成式，要进行版本控制必须联网，要经过中央服务器才能进行版本控制，非常依赖于中央服务器

git为分布式，不用网络就可以进行版本控制，在本机进行版本控制就好

使用git和github的步骤

  1. 安装git

  2. 注册github账号

  3. 关联：命令——ssh-keygen -t rsa -C “注册的邮箱” —>拿到密钥

    ​ 命令——ssh -T git@github.com —>查看是否关联，如果关联上了会出现hi…

  4. 设置贡献者：配置本地提交的用户名和邮箱（可以省略）

    ​ git config –global user.email “你提交分支的时候提交记录里显示的用户邮箱”

    ​ git config –global user.name “提交分支的时候提交记录里显示的用户名”

    ​ git config –global user.email —>查看用户邮箱

    ​ git config –list 查看所有配置项

  5. 创建一个github项目

  6. 把仓库中的代码复制到本地：

    ​ git clone 仓库地址 粘贴：shift+insert或者右击选择粘贴

    ​ 进入本地文件夹：cd 目录 —>进入目录

    ​ 查看磁盘下面的所有目录：ls或者ll

    ​ 返回上一级：cd ..

    ​ 补全：tab键

    ​ 进入项目：出现了master就说明在版本控制

  7. git的三个区：

    1. 工作区（本地）
    2. 暂存区：作为过渡层，避免误操作。保护工作区和版本区，分支处理
    3. 版本区（仓库）

  8. 命令：

    1. 查看状态：git status
    2. 工作区到暂存区：git add 文件名 快捷方式：git add .（一次性将一个或多个文件放到暂存区）
    3. 暂存区到版本区：git commit -m “注释” 快捷方式：直接从工作区到版本区：git commit -a -m “注释”（注意：新建的文件第一次要放到暂存区之后才可以使用这条命令）
    4. 查看版本号：git log，如果git log查看不到版本号，就用git reflog查看已操作的历史纪录
    5. 本地上传到仓库：git push origin master
    6. 清屏：clear
