微信小程序

发表于 2018-06-06

小程序主体部分文件：

主要文件作用：

  1. app.json文件：是对当前小程序的全局配置，包括小程序的所有页面路径（pages:[]）、window界面表现（window:{}）、底部tab（tabBar:{}）、设置网络超时时间（networkTimeout:{}）等配置

    1. pages：数组的第一项代表小程序的初始页面。小程序中增加或者减少页面都需要对pages数组进行修改
    2. window：用于设置小程序的状态栏、导航条、标题、窗口背景色。注意：backgroundColor 是下拉窗口的背景色不是整个页面的背景色，整个页面背景色可以在app.wxss里面设置page{background:’#fff’}
    3. tabBar：底部tab切换。注意：小程序只有首页才有tabBar，跳转到其他页面就没有了

  2. app.js文件：小程序逻辑， App() 函数用来注册一个小程序，接受一个object参数，指定小程序的生命周期函数和全局的数据

  3. app.wxss：小程序公共样式表

小程序页面文件：

  1. wxml文件：页面结构
  2. wxss：页面样式表
  3. js：页面逻辑
  4. json：页面配置（不是必须）

注意：四类文件必须相同名称

注册页面：

Page ( object ) 函数用来注册一个页面，指定页面的初始数据、生命周期函数、事件处理函数，这个就是每个页面的js文件里的格式，相当于一个个vue组件页

  1. data：页面的初始数据
  2. onLoad：监听页面加载
  3. onShow：监听页面显示
  4. onReady：监听页面初次渲染完成
  5. onHide：监听页面隐藏
  6. onUnload：监听页面卸载
  7. onPullDownRefresh：监听用户下拉动作
  8. onReachBottom：页面上拉触底事件的处理函数
  9. onShareAppMessage：用户点击右上角转发
  10. onPageScroll：页面滚动触发事件的处理函数
  11. onTabItemTap：当前是 tab 页时，点击 tab 时触发

  1. 修改数据的时候需要通过 this.setData( { key : value }, callback )，才能改变页面状态

页面路由：

getCurrentPages() 函数用于获取当前页面栈的实例，与vue的当前路由this.$route差不多

文件作用域：

在每个页面的 js 文件中声明的变量和函数只能在当前文件中有效，不同的文件中可以声明相同名字的变量和函数，不会相互影响，通过全局函数 getApp() 可以获取全局的应用实例，如果需要全局的数据可以在 App() 里设置。getApp().globalData

模块化：

把一些公用的代码抽离出来放到一个 js 文件里，通过 exports 或者 module.exports 将该文件暴露出去，使用的时候通过 require(‘相对路径’)，然后就可以调用 js 文件里的方法了，exports 是 module.exports 的引用，随意更改 exports 的指向会造成未知的错误，所以推荐使用 module.exports。

指令：

  1. 数据绑定：

  2. 列表渲染：wx:for=”“ wx:for-item=”自定义的每一项名称” wx:for-index=”自定义的每一项索引” wx:key=”不写会出警告，不会报错”

  3. 条件渲染：wx:if=”“ wx:elif=”“ wx:else

  4. 模板：template组件，通过name属性指定模板的名称，使用is属性声明需要的模板（is=”模板的名称name”），通过data传入模板需要的数据

  5. 事件：

    1. 冒泡事件

    2. 非冒泡事件

    3. 事件绑定可以通过bind或者catch绑定：bindtap、catchtouchstart，bind绑定的事件不会阻止冒泡，catch绑定的事件会阻止冒泡

    4. 事件类型：

      touchstart：手指触摸开始

      touchmove：手指触摸后移动

      touchend：手指触摸结束

      touchcancel：手指触摸被打断，如弹窗

      tap：手指触摸后马上离开，相当于点击

      longpress：长按事件，超过350ms触发，如果指定了这个事件并且出发了，则tap事件不会触发

    5. 事件对象

WXML：

  1. 每个小程序页面的数据都是来自对应 js 文件的Page({})中的data，内容绑定和属性绑定都是双花括号，关键字也需要在双花括号之内，可以在双花括号内进行简单的运算（三元运算、算数运算、逻辑判断、字符串运算等）

自定义组件：

  1. 创建一个components文件夹，里面专门存放组件
  2. 在components文件夹中新建一个目录，跟pages里面的目录一样，有wxml，wxss，js和json文件
  3. 在组件的json文件中写入{“component”:true}
  4. 使用组件：在使用组件的页面的json文件中写入{“usingComponents”:{“navbar”:”../../components/navBar/navBar”}}
  5. 在使用组件的页面的wxml文件中写入组件名称的标签对即可

自定义组件传数据：

  1. 要在自定义组件的js文件的对外属性（properties ）列表中指明传递过来的数据的类型

  2. 在引用组件的wxml页面中通过组件文件中的数据名称等于应用组件传递的数剧
